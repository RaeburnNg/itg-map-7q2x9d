他怎么想(跟他对频很有用)
  • 先定 deliverable / 先量化 —— 还没框定「我们到底要交什么」之前,他不爱往下讨论 idea。带 checklist、带可写出来的清单去找他,推进最快。(6/27·6/28)
  • 结构先于内容 —— 框架不清他动不了,会亲自停下来搭骨架再填;给他散的,他先 restructure。(6/16)
  • outcome-first —— 评一个东西好不好之前,先把「这东西的 outcome 是什么」钉死。(6/28)
  • 受众真相 lens —— 反复把大家从「我们觉得帅」拉回「做生意的人 / 观众看了 get 到什么」。他常说「你看看,我是那个观众,我看了感觉什么?」(6/27·6/28)
  • 直接讲,不解释 —— 作品该直接讲它要做什么,不要绕一圈去解释 / 自证。「我们不要解释,我们要直接讲我们要做什么。」(6/28)
  • 大框架带小框架 + 做减法 —— 用大框架收编小框架来去掉杂讯、降低体感工作量;反复把 video / poster 数量往下砍、主张合并,而不是堆量。「其实我们只是拍三样东西。」(6/27·6/28)
  • picture-first —— 抽象的东西他会先翻成一个画面 / set-piece(三大佬围一桌、空一个位让人「入局」),用具体场景去想。(6/16)
💪
强项(团队可以跟他学的)
  • 系统化 / 框架化 —— 把乱的拆成可复用结构(System Map / skill / SOP)。
  • 创意 / 画面思维 —— picture-first,先看「感觉对不对」,steers 走 Netflix / bad-ass / 有玩味的 tone,而不是「企业很扁」那种。(6/28)
  • idea 表达 logic —— 先讲通用原则 → 再落具体 → 收成一句最强 framing;能 show 就不 tell。
  • 用 AI 的方法(很值得学) —— 给 AI 结构 + context + 真实数据(不让它编)· 当 thinking partner 来回迭代 · 把会议 record 下来喂进去当 input · 把重复的事做成 skill。他正在 build 一个「全队同一个 project workspace + persona」的玩法,让团队的思考越喂越聪明。(6/28)
🔥
怎么带团队 / 怎么激励
  • 不靠钱、靠把人带成长 —— 用「让你被看见 · 给你 own 一块 · 让你成长 · 这件事有 influence · 我们在同一条船」来拉动。(6/28)
  • 信任 + 放手 —— 给信任本身就是激励(「我把这个责任放给你 = 给了你很大的价值」),让你自己跑。(6/28)
  • 高标准本身就是激励 —— 他信你做得到好的;你做出高大上的东西,他会真的 hype。
  • 要团队主动 / aggressive —— 鼓励大家互相提醒、自己约会议、有事就 voice out、随时拉公司任何人来帮想 idea,不要被动等指令。(6/28)

7-Mode —— 怎么「consult Rae」
他自己跑一套 7-mode;团队遇到决策 / 卡住 / 要创意时,可以「带对的东西去 consult 对的 mode」。
Mode你带这个来Rae 怎么跑 → 你学到的
Decision 决策要拍板的事给你 选项+取舍+什么最重要+推荐。学:别只给选项,带推荐。
Stuck 卡住推不动不给 to-do list,一句反问找真 block,再给下一步。学:卡住先找真问题。
Pressure Test 压力测帮我压一压不 validate,直接找漏洞。学:自己先 pressure test 再拿来。
Clarity 厘清一堆乱想法压成一句最强 framing。
Conversation 对话想 brainstorm双向、engage、先问再答。
Create 创作做创意 / 视觉emotion / story / picture-first + 高大上 + 结构带灵魂。学:先想画面+感觉。
Reflect 反思想沉淀听 mode,不给建议除非你要。

怎么跟他合作(实操)
  • 带结构化的东西去:一页图 / checklist / 选项+推荐 → 他决定 + 收口。
  • 别带散的、别只口头讲 —— 写出来的东西要有层次 / 重点放大,不要 flat list(「这样写我们看东西会乱」)。(6/28)
  • 有事就讲、随时 update —— 他最在意的不是你慢,是你「消失 / 沉默」。卡住了 voice out,他来帮你解。(6/28)
  • 先做对东西,再整理排版 —— 资产整理后置,先把内容做对。(6/28)
