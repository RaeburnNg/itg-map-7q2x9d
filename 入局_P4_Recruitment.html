🧱 P4 — 招募漏斗 + 预检硬 gate

这条 pillar 一句话:把「报名 → 筛 → online interview → 预检能否兑现人脉 → 选 6」从机制落成可执行系统。唯一硬 gate = 预检不到能兑现的人脉就不进 6 强 — after-show 兑现的唯一信用防线。
称呼锁:报名进来的叫 企业家(不叫参赛者)· 三位投资人叫 行业大佬(Norman / Tony / Shane)· Jeff = 发起人 / 主持人。本页按 4 个 segment 排:CRM→Database · Form · Online Interview SOP · 预检 gate / 黑名单 / 可兑现人脉池。空 segment 留 🔴 to-build,不删 — form 的空格故意留可见。

③ Online Interview SOP(A 诉求 / B 预检 双栏)To build
~20 强视讯面试 · 双轨记录表 · A = 他想要什么 / B = 我们能不能兑现 · owner Wai Kit
Online interview 两个目的:(1) 看人 — 会不会讲、presentation 能不能撑得起镜头;(2) 拿数据喂预检 — 把 A 栏诉求问清楚,interview 后才能拿去问大佬能不能兑现(B 栏)。所以记录表 双栏:A 诉求 · B 预检 Y/N。
TRACK A
诉求记录(他想要什么)
  • 想见谁 / 想要什么 — 把 form 的诉求问到能验证的具体度(具体到「哪位老板 / 哪个圈 / 哪类拆解」)
  • 突破点 — 他想突破什么 · 节目里能呈现的 drama / arc
  • presentation 能力 — 2 分钟自我介绍试讲、表达、镜头感
  • 记进 Lark 参赛者表 A 栏
TRACK B · GATE
预检记录(我们能不能兑现)
  • interview 后,拿 A 栏诉求去问 Norman / Tony / Shane:「这个人 / 这条资源,找得到吗?帮得到吗?」
  • B = Y → 大佬确认可兑现 → 有资格进 6 强候选
  • B = N → 兑现不了 / ask 太虚 → 不进 6 强(硬 gate)
  • 记进 Lark 参赛者表「人脉可 fulfill」字段 + 备注

  • 双轨记录表模板(A / B 两栏,接 Lark database)
